## 해시

key → value 형태로 데이터를 저장하는 비선형 자료구조

내부적으로는 key에 대해 해시 함수를 통해 고유한 인덱스를 생성하여 빠르게 데이터에 접근

JavaScript에선 Object, Map이 대표적 구현

### 장점

-   빠른 데이터 접근
-   중복 없이 데이터 저장

### 단점

-   순서 보장 x
-   메모리 사용 ⬆️

### 사용 예시

-   사용자 ID → 회원 정보 매핑
-   문자열 등장 횟수 세기
-   중복 제거 (Set도 내부적으로 해시 사용)
-   캐싱 시스템 (key → 데이터 구조)
